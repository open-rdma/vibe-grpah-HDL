# BSV转写自然语言描述要点

> 所属知识库：`prompt_v2.md` | 加载时机：闭环迭代，需要从bsv生成yaml文件时

- 在迭代过程中由原始bsv生成yaml文件时，参照prompt_v2_optimization.md的描述，尽可能使用L0级别，在description中用自然语言描述原有bsv的逻辑。如有任何升级必要，则必须开启新一轮迭代。尽量提升prompt_v2_metrics.md中的AL指标。
- 严禁在任何迭代步骤中采用将原始bsv文件按语法或分词等方式进行拆解后写入yaml文件的行为或者尝试。
- 在转写的过程中，启动一个独立的审计Agent，对转写结果是否足够符合自然语言的描述进行打分，如果得分过低，则重新生成。


- 自然语言不能由任何与BSV语言本身特性相关的描述：
    * 错误示例： TExp#(31)
    * 正确示例： 2的31次方， 将2取31次幂， 31bit位宽可以表达的最大值加1

- 使用自然语言来描述电路时序特性，例如：
    * 将xxxx功能分3拍完成，第一拍做xxx，第二拍做xxx，第三拍做xxx，通过<FIFO/Reg>在相邻两拍之间传递数据。
    * 在第2拍检测A信号，若为真，将B数据放入队列，放入队列的数据将在xxx功能的状态机的xxx状态下取出。
    * 实现一个有N个状态的状态机，其中状态1负责xxxx，状态2负责xxxxx，当满足xxx条件时，从状态xx转换到状态xx。其他所有情况保持当前状态不变。
    * 这里需要一个异步FIFO，从时钟域xxx向时钟域xxx传递信息。FIFO的深度要满足存放xxx条件下产生的数据量。
