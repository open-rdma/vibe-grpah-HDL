## Module Description:
Work Completion (WC) generation module for RDMA Send Queue (SQ) and Receive Queue (RQ) processing pipelines. This module generates IB-style Work Completions that are posted to Completion Queues. It contains two state-machine-driven sub-modules:
1. mkWorkCompGenSQ: Generates Work Completions for the Send Queue. The
   state machine operates in three states: WC_GEN_ST_STOP (idle/reset),
   WC_GEN_ST_NORMAL (normal operation when connection is RTS and non-error),
   WC_GEN_ST_ERR_FLUSH (error flush mode). Normal operation pipeline:
   recvWorkCompGenReqSQ merges WC generation requests from two sources
   (request generator and response handler) into pendingWorkCompQ4SQ;
   genPendingWorkCompSQ converts requests to PendingWorkCompSQ structs
   with the work completion, success status, and a needWorkCompWhenNormal
   flag; waitDmaDoneSQ waits for DMA write completion responses by
   matching PSN values if the WR requires DMA; genWorkCompSQ enqueues
   the final work completion or transitions to error state on failure.
   Error handling: on any non-success WC status, transitions to
   WC_GEN_ST_ERR_FLUSH. In error state, noDmaWaitSQ bypasses DMA wait,
   errFlushSQ flushes all pending WCs with WR_FLUSH_ERR status (handling
   the special case of partial-ACK-then-full-ACK for the same WR by
   skipping the first error WC's corresponding full-ACK flush),
   discardPayloadConRespSQ drains stale DMA responses. Uses
   conflict_free attribute to allow multiple rules to fire concurrently.

2. mkWorkCompGenRQ: Generates Work Completions for the Receive Queue.
   Same three-state machine (STOP/NORMAL/ERR_FLUSH) with similar
   pipeline: recvWorkCompReqRQ classifies incoming requests by RDMA opcode
   (send, write, write-imm, first-or-only, last-or-only) and determines
   if DMA write response is needed; waitDmaDoneRQ waits for DMA if needed
   and enqueues for WC generation (send/write-imm requests generate WC on
   last-or-only fragments); genWorkCompRQ generates the work completion
   or transitions to error state, sending WC status to SQ via
   wcStatusQ4SQ; noDmaWaitRQ and errFlushRQ handle error flushing
   (generates error WC for first-or-only send requests and write-imm
   requests); discardPayloadConRespRQ drains stale DMA responses.

3. Helper functions: genWorkComp4WorkReq constructs a WorkComp struct
   from WorkCompGenReqSQ using opcode mapping, setting fields like id,
   opcode, flags, status, len, pkey, qpn, immDt, rkey2Inv.
   genWorkComp4RecvReq constructs WorkComp for RQ from WorkCompGenReqRQ,
   handling immDt and rkey2Inv from the receive request.

Key types: PendingWorkCompSQ (holds WC gen request, generated work comp, success status, needWorkCompWhenNormal flag), PendingWorkCompRQ (holds WC gen request, maybe work comp, opcode classification booleans, success status, DMA wait flag), WorkCompGenState (STOP/NORMAL/ERR_FLUSH enum).
